# Deploying Predict2U v227

## Recommended: apply the UI patch over the live repository

Use `Predict2U_Responsive_QA_v227_UI_PATCH.zip` over the current live Predict2U repository. Do **not** delete existing files first.

The supplied source snapshot intentionally does not contain the generated live files:

- `data.js`
- `community.js`

Those files contain current fixtures/community runtime data and must remain in the live repository, or be regenerated by the existing workflows.

## Full source bundle

The FULL bundle was rebuilt from `Predict2U_Stable_Recovery_v218_FULL` and then overlaid with the corrected v227 UI. It restores the CSS, JavaScript, PWA, tests, backend and workflow files that were absent from the earlier v224-v226 design-only bundles.

## After deployment

1. Upload the patch without deleting `data.js` or `community.js`.
2. Allow the `predict2u-v227` service worker to activate.
3. Hard refresh once, or clear the old PWA cache if an older header remains visible.
4. Test at least one phone, one tablet/foldable and one desktop browser.
